## Documentation Standards

- Keep docs current with product and engineering decisions.
- Use clear status labels: `Not Started`, `In Progress`, `Blocked`, `In Review`, `Done`.
- Every tracked action should have a clear owner role, deliverable, acceptance criteria, and verification method.
- Public content must avoid secrets, private credentials, paid-license assets without permission, and unverified scientific claims.
- Product, design, engineering, and QA decisions should be traceable through this folder.
