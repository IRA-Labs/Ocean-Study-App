# Contributing

Thanks for helping improve Ocean Study App.

## Workflow

1. Create a branch from `main`.
2. Keep changes focused on one product, content, or engineering goal.
3. Run `npm run check` before opening a pull request.
4. Update docs when product behavior, architecture, QA, or release behavior changes.
5. Use the pull request template and include validation notes.

## Content Standards

- Keep learner-facing content focused on ocean study.
- Make scientific claims source-aware and suitable for introductory learners.
- Do not add private datasets, secrets, paid-license media without permission, or unsupported claims.
- Prefer public-domain, owned, generated, or clearly licensed assets.

## Code Standards

- Use TypeScript and existing Expo Router patterns.
- Prefer reusable components and typed data helpers.
- Keep UI accessible, mobile-first, readable, and image-safe.
- Avoid unrelated refactors in feature or content PRs.
