# Security Policy

## Supported Version

The `main` branch represents the current supported public release state.

## Reporting a Vulnerability

Please report security concerns through GitHub private vulnerability reporting when available, or contact the maintainers through the organization-approved private channel.

Do not open a public issue for secrets, credentials, exploitable vulnerabilities, or private data exposure.

## Security Baseline

- No production secrets should be committed.
- Local environment files must stay out of source control.
- External links should point to reputable ocean science or project resources.
- Dependency remediation should avoid forced downgrades that break the Expo SDK without maintainer review.
